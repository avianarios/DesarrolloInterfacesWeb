Utility-first

Tailwindcss uses a huge amount of small but simple classes with descriptive names (Utility-first approach)

Advantages: reusable classes, easy to read code and mantain it,

HTML code gets bigger, but it is easy to mantain and just by reading the classes name you know what to expect from it

no new classes have to be created for every single component

Class have a recommended order. prettier reorders them (as well as HTML and CSS code)

Some classes have modifiers to show that a cuantity has changed or that a class applies only to certain screens

Tailwindcss is responsive

It provides breakpoints and prefix to identify them

  • It uses a mobile-first approach (Bootstrap does it too) meaning unprefixed classes take effect on all screen sizes, while prefixed ones (like md:flex) take onle at the specified breakpoint and above
  • Do not use sm prefix for mobile, just the unprefixed version
  • class="text-center sm:text-left" will center text on mobile phones and align left at the rest of devices
Breakpoint prefix Minimum width CSS
sm 640px @media (min-width: 640px) { ... }
md 768px @media (min-width: 768px) { ... }
lg 1024px @media (min-width: 1024px) { ... }
xl 1280px @media (min-width: 1280px) { ... }
2xl 1536px @media (min-width: 1536px) { ... }

prefix the utility (class) with the name of the breakpoint followed by colon to apply it only at this breakpoint. (md:w-32 will apply a width of 32 only to md screens, 768px to 1024px)
