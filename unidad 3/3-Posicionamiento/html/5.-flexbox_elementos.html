Flexbox properties acting over flex items

align-self

Align-self allows to define behaviour only for one element in the traverse axis (i.e. the axis perpendicular to the main axis of the flex container). Some values require some empty space for the elements to expand. Values: flex-start, flex-end, center, baseline y stretch

align-self

The base alignment is align-item:center. If not specified, the flex boxes takes all the available space

Third element aligns at the end (align-self:end), 7th at stretch, 8th at start

Grow properties

flex-basis, flex-grow y flex-shrink

flex: a b c is a shortcut for the three properties. If a=number -> equivalent to flex-grow, if a=size -> equivalent to flex-basis if a and b are set, a=grow and b=shrink if it is a number or b=flex-basis if it is a size if a, b and c, then a=grow, b=shrink and c=basis

flex-basis

Defines the initial size of the flexbox on its MAIN AXIS. It may grow due to other properties before being presented to user

If not defined: flex boxes take the minimum space to accomodate their content

If defined: flex boxes are rendered with this size unless not enough room on the container. Anyway, proportions are respected

Boxes 3 and 6 have changed theis size. Wrap is allowed

flex-grow

It specifies the amount of FREE SPACE assigned to the elements within the flex container. Free space is defined as the size of flex container minus the size of all flex items combined AFTER being renderized

Items will first fill the main axis according to flex-basis, then free space is redistributed. Flex-grow will not force items to wrap. If there's no free space, no space is redistributed

item 2 has flex-grown:2 and item 4, flex-grown 3

flex-shrink

The opposite to flex-grow

If the size of all flex items is larger than the flex container, items shrink to fit according to flex-shrink.
